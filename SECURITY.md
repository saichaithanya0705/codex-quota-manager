# Security Policy

## Reporting

If you discover a security issue, do not open a public GitHub issue.

Report it privately to:

- `ysaichaithu2005@gmail.com`

Include:

- A description of the issue
- Affected versions or commit range
- Reproduction steps or proof of concept
- Suggested mitigation if you have one

## Response

The goal is to acknowledge valid reports quickly, reproduce the issue, and ship a fix before public disclosure when possible.

## Scope

Security reports are especially relevant for:

- Credential storage and account switching
- Token refresh and OAuth handling
- Release and publish automation
- Any behavior that could leak auth state or corrupt local Codex/OpenCode credentials
